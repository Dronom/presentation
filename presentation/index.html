HISTORY

SVG developed by the World Wide Web Consortium (W3C) since 1999

W3C is the main international standards organization for the World Wide Web

Among their standards we can find: CSS, DOM, HTML, XML and others

1.1

SVG 1.1 became a W3C Recommendation on 14 January 2003. There is very little difference between SVG 1.1 and SVG 1.0.

1.2

SVG Tiny 1.2 became a W3C Recommendation on 22 December 2008. In this version has been added animation element,"editing-mode" and run-time synchronisation attributes to audio, video and animation.

2.0

SVG 2.0 removes some features of SVG 1.1 and incorporates new features from HTML5:

  • HTML5 features such as translate and data-* attributes have been added.
  • SVG 2.0 removes several font elements such as glyph and altGlyph
  • delete obsolete attributes

SVG 2.0 reached Candidate Recommendation stage on 15 September 2016. The latest draft was released on 13 April 2019.

Why SVG?

Main advantages:

  • Scalability
  • Interactivity
  • Compact file-size
  • Flexibility
  • SEO friendly

Disadvantages:

  • Photo depth
  • Possible printing problems
  • Drawing speed
